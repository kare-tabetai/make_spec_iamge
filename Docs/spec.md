# 企画書生成AIエージェントシステム概要仕様書

## これまでの経緯

ゲームの企画書という創作性の高い部分がAIに置き換わる可能性はどの程度高いかの検証がしたかったのです。
「アイデアとは既存要素の組み合わせである」という有名な格言がすべてなら、組み合わせを高速、大量に行えるAIのほうが圧倒的に有利なはずだと思いました。
そこで、検証のため企画書生成AIエージェントシステムを構築しようとしています。
Difyでプロトタイプを作ってみたがノードベースで設定するのが手間、細かくワークフロー的に設定するよりもエージェントを構築して丸投げするほうが効果が高そうと思いGoogle ADKを使ったマルチエージェントシステムでの実装に変更しようとしています。
Difyのノードのデータを参考にしつつ、Google ADKでエージェントを構築していきます。

## 情報

- Difyで構築したプロトタイプのノード情報はDocs/make_spec_iamge_20262306.ymlに記載しています
- Google ADKについて事前調査したドキュメントはDocs/ADKマルチエージェントシステム構築ガイド.mdにあります
- 良い企画書の書き方については、Docs/ペラ1企画書作成ガイド作成依頼.mdに記載しています
- 使えるGeminiのモデルの情報「[https://ai.google.dev/gemini-api/docs/pricing?hl=ja](https://ai.google.dev/gemini-api/docs/pricing?hl=en)」

## 要求仕様

- Python+Google ADKで実装を行う想定です
- 細かくエージェントの挙動の調整をする必要があるため、まずはシンプルな動作にしてほしいです
- 企画書の情報をマークダウンかjsonでまとめたうえで、それらの情報をまとめてgeminiの画像生成モデルを使ってペラ１企画書を生成してください
- 文章ベースの企画書よりも画像ベースのペラ１企画書の精度を高めることが重要な目標です
- docker環境ですが、一応uvかvenvを使うようにしてください
- ワークスペース直下にAPIキーやトークンが書かれた.envファイルがありますが直接参照しないでください。内容については、ワークスペース直下の.env.exampleを参照してください。環境変数として使う想定です
- 核となるアイデアやキーワード、どのような場面を想定したゲームの企画なのかは引数として文章で与える想定です
- エージェント間でのやり取りはjson形式で行うようにしてください。そのためのjsonスキーマも用意してください
- 一度の実行でN枚のペラ１企画書を生成できるようにしてください。Nはデフォルトで３枚で、それぞれ方向性の違う内容になるようにしてください
- なるべく同じような企画書が生成されないようにUX的に多様性のあるコア体験レベルで異なる内容にしてください
- 初期設計では以下のエージェント構成を想定しています。ただし、必要に応じて、追加・削除・統合を行います
  - web調査エージェント(引数に関連する情報を調べる)
  - ブレインストーミングエージェント
  - コアアイデア検討エージェント
  - アイデア評価エージェント
  - ペラ１画像生成プロンプト生成エージェント
- GUIは初期段階では不要です
- 大まかな初期方針として
  - 引数で与えられた内容をwebで調査
  - 調査した内容と引数のキーワードをもとにブレインストーミング等のアイデア創出手法を用いてアイデアを広げる
  - 広げたアイデアから具体的なゲーム企画のコアアイデアを複数検討する(最低でもNよりも２倍以上多い数)
  - 下記の評価をもとに使えそうなアイデアを抽出
  - 抽出されたアイデアから内容を膨らませる
  - 内容を膨らませたアイデアをペラ１画像生成プロンプトに変換して画像を生成
  - 生成した画像をpng形式で保存、膨らませたアイデアの情報もテキストorマークダウンorjsonとして保存
- ペラ１画像の解像度は0.5Kを選択してください
- 各エージェントの推論内容や、推論の結果は実行ごと、エージェントごとにログとして出力するようにしてください

### 使用モデル

- テスト用で安いモデルを使うモードと本番用の高いモデルを使うモードを切り替えられるようにしてください
- 画像生成モデルはトークン使用料が高いので、頻繁に使用しないでください
  - 画像生成モデルは、最後にのみ必要なはずです
- 推論には、下記のモデルを使うようにしてください
  - 本番用: 「gemini-3-pro-preview」
  - 検証用: 「gemini-2.0-flash-lite」
- 画像生成モデルは、下記のモデルを使うようにしてください
  - 本番用、検証用: 「gemini-3.1-flash-image-preview」

### 企画書内容の評価

- 斬新さ
- 明確さ、わかりやすさ
- ぱっと見て面白そうか
- 実現可能か

という評価軸でプロのゲームデザイナーが企画書を評価する想定で作成してください

### ペラ1企画書自体の評価

上記の企画書内容の評価軸に加えて、以下の評価軸も追加します

- アートスタイルが企画内容に一致しているか
- 文章があまりにも多すぎないか
- レイアウトが見やすいか
- 必要な情報のみを明示できているか

上記の評価軸でプロのゲームデザイナーが企画書を評価する想定で作成してください
