# 企画書の面白さ・多様性を向上させる改善案

## 背景・課題

ゲーム企画書生成パイプラインの出力するマークダウン企画書について、以下の問題がある：

1. **多様性の不足**: 同一トピックで繰り返し実行すると似たアイデアに収束しやすい
2. **面白さの不足**: 企画書に「これは面白そう！」と直感的に思わせる要素が弱い

既存パイプライン構造は維持しつつ、プロンプト改善・フィールド追加・マークダウンフォーマット刷新・パイプラインロジック改善で対応する。

---

## 改善一覧

### Phase 1: プロンプト改善（既存コードへの小規模追記のみ）

#### 改善1: CoreIdeaAgentでcross_connections活用

- **概要**: BrainstormMergeAgentが抽出した`cross_connections`（異種手法間の意外な組み合わせ）を、CoreIdeaAgentのプロンプトで明示的に活用する指示を追加
- **効果**: 5手法をまたいだ「化学反応」的アイデアが増え、単一手法では出ないユニークな企画が生まれる
- **難易度**: 低（プロンプト数行の追加）
- **対象ファイル**: `src/game_pitch_agent/agents/core_idea.py`

#### 改善2: ExpansionAgentへのFew-shot例示追加

- **概要**: 高品質な架空の企画書を1例、プロンプト内に追加。キャッチコピー・concept・game_cycle・core_mechanicの「良い品質レベル」を具体的に示す
- **効果**: LLMがFew-shotに引っ張られる性質を利用し、出力品質の底上げと安定化。特にキャッチコピーの質が向上
- **難易度**: 低（プロンプト追加のみ）
- **対象ファイル**: `src/game_pitch_agent/agents/expansion.py`

---

### Phase 2: スキーマ拡張（新フィールド4つ追加）

#### 改善3: 「プレイシーン描写」フィールド追加

- **概要**: `ExpandedPitch`に`play_scene`(string)を追加。「読者がゲームプレイを脳内再生できる30-50文字の具体的ワンシーン」を生成
- **効果**: 企画書を読んだ人が「こういうゲームか！」と直感的に理解できる
- **対象ファイル**: `schemas/models.py`, `agents/expansion.py`, `main.py`

#### 改善4: 「エレベーターピッチ」フィールド追加

- **概要**: `ExpandedPitch`に`elevator_pitch`(string)を追加。「[作品A] meets [作品B]」形式の一文で本質を伝える
- **効果**: ゲーム業界定番のピッチ形式で、読者の興味を即座に引ける
- **対象ファイル**: `schemas/models.py`, `agents/expansion.py`, `main.py`

#### 改善5: 「感情曲線」フィールド追加

- **概要**: `ExpandedPitch`に`emotional_curve`(string)を追加。プレイ開始〜1時間の感情変化を記述（例: 困惑→発見→挫折→工夫→達成感）
- **効果**: コンセプトとメカニクスの整合性が読み手に検証可能になり、企画書の説得力が増す
- **対象ファイル**: `schemas/models.py`, `agents/expansion.py`, `main.py`

#### 改善6: 「ターゲットプレイヤー」フィールド復活

- **概要**: `ExpandedPitch`に`target_player`(string)を追加。CoreGameIdeaには存在するが展開時に落ちている情報を復活させる
- **効果**: ターゲットが明確な企画書は説得力が高い
- **対象ファイル**: `schemas/models.py`, `agents/expansion.py`, `main.py`

---

### Phase 3: マークダウン出力の構成刷新

#### 改善7: build_markdown()の全面リフォーマット

- **概要**: 現在の平板な構成を、読者を引き込む物語的構成に刷新
  - タイトル + キャッチコピー + エレベーターピッチを冒頭に集約
  - プレイシーン描写を冒頭セクションに配置
  - ジャンル/プラットフォーム等の基本情報をテーブル形式に
  - ゲームサイクルをテキストフローチャートで循環表現
  - USPをボックス引用で目立たせる
  - 感情曲線・ターゲットプレイヤーの適切な配置
- **効果**: 同じ内容でも「読みたくなる」企画書に変わる
- **難易度**: 低（1関数の書き換え）
- **対象ファイル**: `src/game_pitch_agent/main.py` の `build_markdown()`

---

### Phase 4: パイプラインロジック改善

#### 改善8: ブレスト段階への「ランダム制約カード」注入

- **概要**: パイプライン実行時にランダムな制約をSession Stateに注入し、各ブレストエージェントが異なる制約下でアイデアを発散する仕組み
  - **ジャンル制約**（例: RPG視点で考えよ）
  - **感情制約**（例: 懐かしさをコアに）
  - **メカニクス制約**（例: 物理演算不使用）
  - **ターゲット制約**（例: 60歳以上向け）
- **効果**: 同一トピックでも毎回異なるアイデアが出る。多様性の劇的改善
- **難易度**: 中（制約定義ファイル新規作成 + パイプライン・プロンプト修正）
- **対象ファイル**: `pipeline.py`, `agents/brainstorm.py`, 新規`constraints.py`

## 実装方針

- 各Phaseは独立して実装・テスト可能
- Phase 1→2→3→4の順で段階的に実装する
- 各Phase完了時にテスト生成を行い、出力品質を確認する
- 既存パイプライン構造（12ステップSequentialAgent）は変更しない
